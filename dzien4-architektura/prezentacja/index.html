MYSPOT: REZERWACJA MIEJSC PARKINGOWYCH

wewnętrzna apka pracownicza
Wewnętrzna aplikacja dla pracowników firmy: każdy rezerwuje miejsce w garażu na konkretny dzień, a system pilnuje reguł biznesowych.
  • Rezerwacje planowane tygodniowo: na bieżący tydzień pracy.
  • Limit rezerwacji zależy od stanowiska (JobTitle): Boss, Manager i Regular mają różne uprawnienia.
  • Na jednym miejscu parkuje 1 samochód albo 2 motocykle.
  • Sprzątanie garażu wymaga pustych miejsc: rezerwacje na ten dzień są zdejmowane.
  • Za złe parkowanie grozi tymczasowy zakaz rezerwacji.
🔑 Prosta domena z realnymi regułami: dzięki temu moduły mają różną złożoność i różne style.

JEDNA APLIKACJA, JEDNA JEDNOSTKA WDROŻENIOWA

var builder = WebApplication.CreateBuilder(args);
builder.Host.ConfigureModules().UseLogging();

// refleksja: host odnajduje moduły w DLL-kach (o tym za chwilę)
var assemblies = ModuleLoader.LoadAssemblies(builder.Configuration, "MySpot.Modules.");
var modules = ModuleLoader.LoadModules(assemblies);

builder.Services.AddModularInfrastructure(builder.Configuration, assemblies, modules);
foreach (var module in modules)
    module.Register(builder.Services, builder.Configuration); // usługi do DI

var app = builder.Build();
app.UseModularInfrastructure();
foreach (var module in modules)
    module.Use(app);    // middleware modułu do pipeline'u

foreach (var module in modules)
    module.Expose(app); // endpointy modułu

app.Run();

CZYM JEST REFLEKSJA

NOWOŚĆ · REFLEKSJA
Refleksja to mechanizm, dzięki któremu program ogląda własne typy w trakcie działania. Klasa, metoda czy interfejs to w .NET obiekty (Type), które można wyliczyć, przefiltrować i utworzyć z nich instancje bez new zapisanego w kodzie.
Type type = typeof(AvailabilityModule);           // typ jako obiekt
Type[] types = assembly.GetTypes();               // wszystkie typy z DLL-ki
object instance = Activator.CreateInstance(type); // instancja bez new
🔑 Kod, który czyta kod. Dzięki temu host znajdzie moduły, których nie zna z nazwy.
{ 11 / 43 }
Jedyna prawdziwa nowość dzisiejszego dnia. Do tej pory typy znaliście w czasie kompilacji; refleksja pozwala pracować na nich w runtime, jak na zwykłych danych. Zaraz zobaczycie, po co to modularnemu monolitowi.

MODULELOADER: REFLEKSJA W AKCJI

NOWOŚĆ · REFLEKSJA
public static IList<IModule> LoadModules(IEnumerable<Assembly> assemblies)
    => assemblies
        .SelectMany(x => x.GetTypes())                                     // wylicz wszystkie typy
        .Where(x => typeof(IModule).IsAssignableFrom(x) && !x.IsInterface) // przefiltruj po interfejsie
        .OrderBy(x => x.Name)
        .Select(Activator.CreateInstance)                                  // utwórz instancje bez new
        .Cast<IModule>()
        .ToList();
🔑 Dokładnie ten mechanizm ze slajdu obok: wyliczamy typy, filtrujemy po interfejsie, tworzymy instancje.
{ 12 / 43 }
Siedem linii, a to one czynią monolit modularnym: host nie ma w kodzie referencji do żadnego modułu z nazwy. Nowy moduł wystarczy dodać do solucji, loader sam go znajdzie. LINQ z Dnia 1 plus refleksja sprzed chwili.

APPLICATIONPARTMANAGER: DRUGA POŁOWA WYŁĄCZNIKA

MVC składa kontrolery z części aplikacji (application parts), zwykle jedna część to jedno assembly. ApplicationPartManager pozwala tą listą zarządzać w runtime: nawet gdy DLL modułu jest załadowana, jego kontrolery znikają z aplikacji.
NOWOŚĆ · APPLICATIONPARTMANAGER
// Shared.Infrastructure/Extensions.cs
services.AddControllers()
    .ConfigureApplicationPartManager(manager =>
    {
        var removedParts = new List<ApplicationPart>();
        foreach (var disabledModule in disabledModules) // lista z poprzedniego slajdu
        {
            removedParts.AddRange(manager.ApplicationParts.Where(x =>
                x.Name.Contains(disabledModule, StringComparison.InvariantCultureIgnoreCase)));
        }
        foreach (var part in removedParts)
        {
            manager.ApplicationParts.Remove(part); // kontrolery wyłączonego modułu przestają istnieć
        }
        manager.FeatureProviders.Add(new InternalControllerFeatureProvider()); // widzi też internal
    });
🔑 LoadAssemblies pomija DLL, ApplicationPartManager sprząta po stronie MVC: wyłączony moduł znika razem z endpointami.
{ 14 / 43 }
Flaga enabled ma dwie połowy. LoadAssemblies nie załaduje DLL wyłączonego modułu, ale część modułów i tak wjeżdża do procesu przez referencje projektowe. Wtedy wkracza ApplicationPartManager: usuwa części aplikacji wyłączonych modułów, więc MVC nie zobaczy ich kontrolerów. Bonus: InternalControllerFeatureProvider pozwala MVC odkrywać kontrolery internal, czyli te schowane wewnątrz modułów.

internal sealed class ParkingSpotsService : IParkingSpotsService
{
    private readonly ParkingSpotsDbContext _context;
    // ... zależności dostarcza kontener DI

    public async Task AddAsync(ParkingSpot parkingSpot)
    {
        await _context.ParkingSpots.AddAsync(parkingSpot);
        await _context.SaveChangesAsync(); // zapis przez EF Core
    }

    public async Task DeleteAsync(Guid parkingSpotId)
    {
        var parkingSpot = await _context.ParkingSpots
            .SingleOrDefaultAsync(x => x.Id == parkingSpotId);
        if (parkingSpot is null)
            throw new ParkingSpotNotFoundException(parkingSpotId);

        _context.ParkingSpots.Remove(parkingSpot);
        await _context.SaveChangesAsync();
    }
}

// znacznik: komenda to zwykły obiekt
public interface ICommand : IMessage { }

public interface ICommandHandler<in TCommand> where TCommand : class, ICommand
{
    Task HandleAsync(TCommand command, CancellationToken cancellationToken = default);
}
DZIEŃ 2 · SERVICE PROVIDER
public sealed class CommandDispatcher : ICommandDispatcher
{
    private readonly IServiceProvider _serviceProvider;

    public CommandDispatcher(IServiceProvider serviceProvider)
        => _serviceProvider = serviceProvider;

    public async Task SendAsync<TCommand>(TCommand command,
        CancellationToken cancellationToken = default) where TCommand : class, ICommand
    {
        using var scope = _serviceProvider.CreateScope();
        var handler = scope.ServiceProvider.GetRequiredService<ICommandHandler<TCommand>>();
        await handler.HandleAsync(command, cancellationToken);
    }

Rejestr ścieżek (ModuleRegistry) mapuje string na akcję modułu docelowego. Serializacja JSON rozprzęga typy: każdy moduł operuje na własnej kopii danych. Wzorzec znany z systemów rozproszonych (routing po ścieżce), tylko wykonywany w jednym procesie.
public async Task<TResult> SendAsync<TResult>(string path, object request,
    CancellationToken cancellationToken = default) where TResult : class
{
    var registration = _moduleRegistry.GetRequestRegistration(path);
    if (registration is null)
        throw new InvalidOperationException($"No action has been defined for path: '{path}'.");

    // translacja: obiekt nadawcy -> JSON -> typ odbiorcy
    var receiverRequest = TranslateType(request, registration.RequestType);
    var result = await registration.Action(receiverRequest, cancellationToken);

    return result is null ? null : TranslateType<TResult>(result);
}

private object TranslateType(object value, Type type)
    => _moduleSerializer.Deserialize(_moduleSerializer.Serialize(value), type);
⚠️ Moduł woła moduł przez rejestr i translację typów, NIGDY przez wspólną bazę. Baza jako kanał komunikacji = rozproszony monolit.
{ 30 / 43 }

public record ParkingSpotCreated(Guid ParkingSpotId) : IEvent;

public async Task PublishAsync(object message, CancellationToken cancellationToken = default)
{
    var key = message.GetType().Name;
    var registrations = _moduleRegistry.GetBroadcastRegistrations(key)
        .Where(r => r.ReceiverType != message.GetType());

    var tasks = new List<Task>();
    foreach (var registration in registrations)
    {
        // każdy odbiorca dostaje własną kopię: serialize -> deserialize
        var receiverMessage = TranslateType(message, registration.ReceiverType);
        tasks.Add(registration.Action(receiverMessage, cancellationToken));
    }
    await Task.WhenAll(tasks);
}

Wewnątrz procesu
Wywołanie modułu albo działa, albo od razu rzuca wyjątek. Odporność zbędna.

Timeout
Limit czasu na pojedynczą próbę i na całość wywołania.
